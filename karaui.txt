    <pre id="pt">## KARA — Master UI Design System Prompt
## Apply this design language to EVERY screen in the app without exception.

---

### Brand identity
- App name: kara
- Logo: lowercase "kara" in a clean serif or rounded sans, followed immediately by a ✦ golden sparkle glyph (not an emoji — use the Unicode character ✦ or an SVG star)
- Tagline register: short, poetic, two-word or three-word lines ("Sing. Feel. Express.", "Your stage. Your song.")

---

### Global color palette — use these exact values everywhere

| Token            | Hex       | Usage                                      |
|------------------|-----------|--------------------------------------------|
| --bg             | #F5F0E8   | Page background — warm parchment           |
| --surface        | #EDE8DE   | Card surfaces, nav bar background          |
| --border         | #C8B89A   | Subtle borders, dividers                   |
| --text-primary   | #1A1A1A   | All headings and body text                 |
| --text-secondary | #6B6560   | Captions, labels, secondary info           |
| --cta-bg         | #1A1A1A   | Primary CTA button fill                    |
| --cta-text       | #F5F0E8   | Text on primary CTA button                 |
| --star           | #F4C24B   | ✦ sparkle icon accent color               |
| --white          | #FFFFFF   | Album card backgrounds, modal surfaces     |

NEVER use pure white (#FFFFFF) as a page background. NEVER use blue, purple, or any saturated accent color unless it comes from album art being displayed.

---

### Typography

Headings (hero, section titles):
- Font: A serif typeface (e.g. Playfair Display, Lora, or Georgia as fallback)
- Weight: 700–800 (bold/extrabold)
- Size: 48–72px for hero, 28–36px for section titles
- Color: #1A1A1A
- Line height: 1.1–1.2 (tight, editorial)
- Letter spacing: -0.02em (slightly condensed)

UI text (nav, labels, captions, buttons):
- Font: System sans-serif (-apple-system, Inter, or similar)
- Weight: 400 regular / 500 medium only
- Size: 13–16px
- Color: #1A1A1A primary, #6B6560 secondary

Pill / badge text:
- Font: sans-serif, weight 400, size 13px
- Always sentence case. Never ALL CAPS.

---

### Component library — apply consistently to ALL screens

#### Navigation bar
- Background: #F5F0E8 (same as page — no separate bar color, no border)
- Layout: logo left, search center (pill shape, #EDE8DE fill, #C8B89A border), auth actions right
- Search pill: border-radius 100px, height 40px, icon + placeholder text "#6B6560", no visible focus glow — just border color shift
- Buttons: "Log in" = text-only, "Sign up" = filled #1A1A1A pill with #F5F0E8 text, border-radius 100px

#### Primary CTA button
- Background: #1A1A1A
- Text: #F5F0E8, sans-serif, 15px, weight 500
- Border-radius: 100px (fully rounded pill)
- Padding: 14px 28px
- Arrow icon: → appended inline, same color
- Hover state: slight scale(1.02), background #2C2C2A

#### Feature icon pills (below hero CTA)
- Layout: row of 3, centered
- Each: circular icon (40px, #EDE8DE bg, 1px #C8B89A border) above a 13px label in #6B6560
- Icons: outline style, 20px, color #6B6560

#### Album art cards (the SIGNATURE visual element)
- Shape: rounded square, border-radius 20px, with subtle white border (1px #FFFFFF40)
- Sizes: vary — large ~200px, medium ~140px, small ~90px
- Rotation: each card tilted randomly between -18deg and +18deg
- Position: absolutely placed, scattered across the hero background
- Depth effect: cards "further away" get opacity 0.5–0.7 AND a CSS blur(2–4px). Close cards are sharp and full opacity.
- DO NOT use a grid for album cards — they must look organically scattered
- Implementation: use absolute positioning with hardcoded left/top percentages per card, each with its own rotation and blur
- The cards bleed off screen edges intentionally

#### Search / input fields
- Background: #EDE8DE
- Border: 0.5px solid #C8B89A
- Border-radius: 100px
- Height: 44px
- Placeholder: #6B6560
- Focus: border-color #1A1A1A, no glow/shadow
- Icon: search glyph left-padded, 16px, color #6B6560

#### Cards (song, album, playlist, result)
- Background: #FFFFFF
- Border-radius: 16px
- Border: 0.5px solid #E0D8CC
- Padding: 12px
- Hover: transform translateY(-2px), border-color #C8B89A
- No drop shadows — depth comes from the background contrast only

#### Modal / overlay panels
- Background: #FFFFFF
- Border-radius: 24px
- Border: 0.5px solid #E0D8CC
- Backdrop: rgba(26,26,26,0.4) behind modal
- Close button: top-right, 32px circle, #EDE8DE bg, #1A1A1A × icon

#### Bottom sheet / player bar
- Background: #1A1A1A (inverted — this is the ONLY dark surface)
- Text: #F5F0E8
- Border-radius: 20px 20px 0 0
- Controls: white icons at 24px, progress bar in #6B6560 with #F5F0E8 filled portion

---

### Screen-by-screen application rules

Apply ALL of the above to EVERY screen. Specific guidance per screen:

**Landing / home page**
- Full-viewport hero with scattered album art cards (absolute positioned, rotated, depth-blurred)
- Centered hero text: serif, 64px, "#1A1A1A", two short punchy lines
- Tagline badge above headline: pill with ✦ icon + 2–3 word tagline, #EDE8DE bg, #6B6560 text
- Single CTA button below headline
- Three feature pills below CTA (icon + label)

**Search results page**
- Same navbar, same #F5F0E8 background
- Results in a clean 2-col or 3-col card grid
- No album-card scatter — this screen is functional, not decorative
- Active/playing card: border-color #1A1A1A (2px), slight scale(1.01)

**Song / karaoke player page**
- Background: #F5F0E8 (never go dark except for the bottom player bar)
- Large centered album art (256px, border-radius 20px, no rotation)
- Lyrics display: centered, large serif for current line, smaller sans for next lines, color fades from #1A1A1A (active) to #C8B89A (upcoming)
- Pitch visualization sits BELOW or ABOVE the lyrics line — words plotted by Y-position (Hz → pixels), connected by diagonal SVG arrows (teal #1D9E75 = rising, coral #D85A30 = falling, gray #888780 = flat)
- Player controls: icon row, centered, #1A1A1A icons on #F5F0E8 bg
- Bottom player bar: #1A1A1A strip with song info + playback

**Profile / settings page**
- Same parchment background
- Avatar: large circle, 80px, with user initials or photo
- Stats row: 3 metric cards in #EDE8DE bg (songs sung, streak, top genre)
- Settings items: full-width rows with label left, control right, divided by 0.5px #E0D8CC lines

**Trending / discover page**
- Section title: serif 28px
- Horizontal scroll rows for genre categories — pills in #EDE8DE, active pill in #1A1A1A/#F5F0E8
- Song cards in a 2-col grid

---

### Things to NEVER do
- Never use blue, purple, green, or any saturated color as a UI color (only from album art)
- Never use pure white (#FFF) as a page background
- Never use gradients on backgrounds or buttons
- Never use drop shadows (box-shadow with x/y offset) — depth is from background contrast only
- Never use ALL CAPS text
- Never use a dark background except the player bottom bar
- Never place album cards in a grid — they must be scattered and rotated
- Never use emoji in the UI — use the ✦ Unicode glyph or Tabler outline icons only
- Never use border-radius below 12px on cards or 8px on small elements

---
